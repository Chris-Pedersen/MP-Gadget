6 Description
7 -----------
8 
9 This version of Gadget is derived from main P-Gadget / Gadget-2. It is
10 the source code used to run the BlueTides simulation
11 (<http://bluetides-project.org>).
12 
13 The infrastructure is heavily reworked. As a summary:
14 
15 - A better PM solver for long range force with Pencil FFT.
16 - A better Tree solver with faster threading and less redundant code.
17 - A better Domain decomposition that scales to half a million cores.
18 - A easier to use IO module with a Python binding.
19 - A more intuitive parameter file parser with schema and docstrings.
20 - A cleaner code base with less conditional compilation flags.
21 
22 Physics models:
23 
24 - Pressure Entropy SPH and Density Entropy SPH
25 - Radiation background in the expansion
26 - Generic tracer particle seeding
27 - Various wind feedback and blackhole feedback models
28 - Various star formation criteria (need a License from Phil Hopkins
29  before enabling)
30 - Primordial and metal cooling
31 - Fluctuating UV background

33 Installation
34 ------------
35 
36 First time users:
37 
38 ``` {.sourceCode .bash}
39 git clone https://github.com/bluetides-project/MP-Gadget
40 # or git clone https://github.com/rainwoodman/MP-Gadget-private
41 cd MP-Gadget
42 
43 bash bootstrap.sh
44 ```
45 
46 We will need gsl. On HPC systems with the modules command, usually it
47 can be loaded with
48 
49 ``` {.sourceCode .bash}
50 module load gsl
51 
52 env | grep GSL # check if GSL path is reasonable
53 ```
54 
55 On a common PC/Linux system, refer to your package vendor how to install
56 gsl and gsl-devel.
57 
58 Copy Options.mk.example to Options.mk
59 
60 ``` {.sourceCode .bash}
61 cp Options.mk.example Options.mk
62 ```
63 
64 Edit Options.mk
65 
66 1. Set GSL flags according to the environment variables. On coma,
67  use Options.mk.example.coma.
68 2. Tweak the compilation options for 'features'. We are in the process
69  of cleaning this up. Most options are tricky and undocumented, as
70  Gadget. The defaults shall work for most cases; it enables
71  Pressure-Entropy SPH and Blackhole, Cooling and SFR. To run a N-Body
72  sim, use IC files with no gas particles.
73 3. For off-tree build, set DESTDIR in Options.mk; the default target is
74  in build/
75 
76 Now we are ready to build
77 
78 ``` {.sourceCode .bash}
79 make -j 8
80 ```
81 
82 It takes some time to build pfft, one of the bundled dependencies. Other
83 libraries are bigfile and mp-sort, which are written by me and quick to
84 build.
85 
86 In the end, we will have 2 binaries:
87 
88 ``` {.sourceCode .}
89 ls build/MP-Gadget build/MP-GenIC
90 ```
91 
92 1. MP-Gadget is the main simulation program.
93 2. MP-GenIC is the initial condition generator.

95 GLIBC 2.22
96 ----------
97 
98 Cray updated their GLIBC to 2.22+ recently. A good move but it happens
99 to be a buggy version of GLIBC:
100 <https://sourceware.org/bugzilla/show_bug.cgi?id=19590> causing
101 non-existing symbols like \_ZGVcN4v\_\_\_log\_finite. Adding
102 -lmvec -lmvec\_nonshared to GSL\_LIBS works around the issue.

114 OpenMP Complication
115 -------------------
116 
117 When OpenMP is switched from on to off or off to on, all of the
118 dependencies needs to be recompiled. This can be achived by removing all
119 files in depends/lib.
120 
121 Otherwise symbols related to OpenMP in PFFT may be missing.
122 
123 Always enable OpenMP.
124 
125 IO Format
126 ---------
127 
128 The snapshot is in bigfile format. For data analysis in Python, use
129 
130 ``` {.sourceCode .bash}
131 pip install bigfile
132 ```
133 
134 Refer to <https://github.com/rainwoodman/bigfile> for usage.
135 
136 Otherwise directly open the blocks with Fortran or C, noting the
137 data-type information and attributes in header and attrs files (in plain
138 text)
