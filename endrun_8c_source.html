17  * function based on xorg_backtrace_pstack; extracted from xorg-server/os/backtrace.c
18  *
19  * an external tool is spawn to investigate the current stack of
20  * the crashing process. there are different opinions about
21  * calling fork in a signal handlers.
22  * But we are already crashing anyways if we land in a signal.
23  *
24  * if no external tool is found we fallback to glibc's backtrace.
25  *
26  * */

55  * causing random failures in execle. */
